# Crayotter Workbench GUI V1

## 1. 这次新增了什么

在 Phase A 后端基础上，这一版新增了一个由后端直接托管的本地工作台页面。

新增文件：

- `app/frontend/index.html`
- `app/frontend/styles.css`
- `app/frontend/app.js`

扩展能力：

- 后端现在会托管 `/ui/`
- 页面支持查看服务状态
- 页面支持编辑默认配置
- 页面支持创建 `demo` / `agent` 任务
- 页面支持查看任务历史
- 页面支持实时显示任务事件流
- 页面支持查看任务详情
- 页面支持浏览任务产物并做简单预览

## 2. 如何启动新版工作台

注意：

- 如果你之前已经启动过旧版后端，必须先停止旧进程
- 然后重新启动，才能加载现在这版新代码

启动命令：

```powershell
python script/run_backend.py --host 127.0.0.1 --port 8765
```

启动后控制台会打印：

```text
Crayotter backend listening on http://127.0.0.1:8765
Crayotter workbench available at http://127.0.0.1:8765/ui/
```

然后在浏览器打开：

```text
http://127.0.0.1:8765/ui/
```

## 3. 工作台能做什么

### 左侧

- 发起任务
- 选择模式
- 编辑默认 API 配置
- 查看任务列表

### 中间

- 查看任务主状态
- 查看流式事件
- 请求取消当前任务

### 右侧

- 查看任务详情
- 查看产物列表
- 预览文本或视频产物

## 4. 推荐验收顺序

### 4.1 验收页面能打开

浏览器打开：

```text
http://127.0.0.1:8765/ui/
```

验收标准：

- 页面能正常加载
- 顶部显示 `Crayotter Local Workbench`
- 左侧有任务输入区和配置区
- 中间有执行流面板
- 右侧有任务详情和产物面板

### 4.2 验收 Demo 模式

在页面左上角：

1. 输入任务，例如 `demo acceptance run`
2. 保持模式为 `Demo 验收模式`
3. 点击 `开始执行`

验收标准：

- 左侧任务列表新增一条任务
- 中间执行流开始滚动显示事件
- 任务状态从 `running` 变成 `completed`
- 右侧出现产物列表
- 文本产物可预览

### 4.3 验收历史任务恢复

刷新页面，或者重启后端后再打开 `/ui/`

验收标准：

- 之前跑过的任务仍能出现在任务列表中
- 可以点击查看任务详情
- 可以查看之前任务的事件和产物

### 4.4 验收配置保存

在左侧配置区填写：

- API Key
- Base URL
- 文本模型
- 视频模型
- TTS 模型

点击 `保存配置`

验收标准：

- 页面出现“配置已保存”
- 本地 `app_state/config.json` 内容更新

### 4.5 验收 Agent 模式

在左侧配置区先写入真实配置，然后：

1. 任务模式切换成 `真实 Agent 模式`
2. 输入真实任务
3. 点击开始

验收标准：

- 状态进入 `running`
- 中间事件流持续出现结构化事件
- 完成后右侧产物里可看到输出文件

## 5. 当前已知限制

- 这还是本地网页工作台，不是 Electron 桌面壳
- 当前任务仍然是单任务串行
- `agent` 模式的强制取消还不彻底
- 视频产物只做了基础预览
- 还没有上传本地素材、阶段重跑、连接测试等更完整的交互

## 6. 这版通过意味着什么

如果这版 GUI 验收通过，说明下面这些关键路径已经跑通了：

- 后端服务
- 配置存储
- 任务调度
- 事件流
- 历史任务恢复
- 浏览器工作台
- 产物查看

这就意味着下一阶段可以直接继续往真正的桌面应用演进，而不是再回头补基础设施。

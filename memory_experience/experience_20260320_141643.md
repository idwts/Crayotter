# Skills: Video Editing Agent Experience
## Context
- **user_request**: 剪辑一个一分钟的城市夜景与通勤节奏混剪，强调镜头流动与音画同步，旁白简洁有节奏感。
- **key_result**: 成功生成 `city_night_rhythm_final.mp4` (60.3s, 1080p)。通过“宏观→微观→升华→余韵”的叙事结构，实现了从神秘夜景到未来赛博朋克，再回归温情归途的情感曲线。旁白采用 7 段式分段配音，覆盖约 27% 时长，有效利用留白增强画面表现力，音画同步率达标。

## Tool Usage Skills
- **素材筛选策略 (`search_bilibili_video` + `rank_video_candidates`)**:
  - **最佳实践**: 采用“多关键词组合”（如：城市夜景 + 通勤 + 延时摄影 + 赛博朋克）进行分页搜索，随后利用 AI 评分工具对画面流动性、夜景质量、节奏感进行二次筛选。
  - **规避模式**: 避免仅依赖单一关键词导致素材同质化（如全是静态夜景）。
  - **执行要点**: 设置 `top_k=8` 下载最优视频，确保素材库包含不同视角（地面、高架、空中）和不同时间维度（日落、深夜、黎明）。

- **智能裁剪与分段 (`cut_video`)**:
  - **最佳实践**: 基于 `analyze_video` 提取的场景切换点，进行非均匀裁剪。例如，高潮部分（赛博朋克飞行汽车）采用连续三段连剪（5s+5s+5s），而叙事铺垫部分采用长镜头（5s+5s）。
  - **规避模式**: 避免机械式等长裁剪（如全部切为 5s），这会破坏叙事节奏。
  - **执行要点**: 裁剪时需预留转场时间（通常 0.3s-0.8s），确保 `merge_videos` 时总时长可控。

- **转场规划 (`plan_transition_timeline` + `add_transition`)**:
  - **最佳实践**: 根据情绪曲线动态选择转场。
    - 开场/收尾：`crossfade` (0.5s) 或 `fade_through_black` (0.5s) 营造呼吸感。
    - 节奏加速段：`zoom_in` (0.4s) 或硬切，增强冲击力。
    - 场景突变（现实→未来）：`fade_through_black` (0.3s) 制造时空跳跃感。
  - **规避模式**: 全片使用同一种转场（如全用淡入淡出），导致视觉疲劳。

- **旁白同步 (`align_narration_to_timeline` + `add_narration_segments`)**:
  - **最佳实践**: 采用“文案驱动时长”的反向校准机制。先确定文案字数，计算理论时长，再微调视频片段起止点以匹配。
  - **关键修正**: 首次尝试出现“旁白明显短于分配时段”警告（如 2.16s/5.00s），系统自动触发二次迭代，增加文案字数（从“城市苏醒”改为“整座城市缓缓苏醒”），使实际旁白时长（16.4s）与目标槽位（34.9s）的覆盖率提升至 27%，符合“留白”策略。
  - **规避模式**: 强行拉长旁白语速或压缩视频片段导致画面卡顿。

## Editing Workflow Skills
- **叙事结构构建**:
  - **四段式法则**: 严格遵循 `开场 (0-10s)` 建立氛围 -> `发展 (10-40s)` 铺陈节奏 -> `高潮 (40-55s)` 视觉爆发 -> `收尾 (55-60s)` 情感升华。
  - **镜头流动逻辑**: 确保镜头运动方向的一致性（如：从地面仰视→高架平视→空中俯视），避免视觉跳跃。

- **节奏控制 (Pacing)**:
  - **快慢对比**: 通勤段（地铁、路口）采用快速剪辑（平均 3-5s/镜），配合快节奏旁白；开场与收尾采用慢速长镜头（5-8s/镜），配合留白。
  - **音画卡点**: 旁白重音（如“奔赴”、“看见未来”）必须落在画面切换点或运动高潮点（如车流光轨最亮处、飞行汽车起飞瞬间）。

- **音频与字幕匹配**:
  - **字体规范**: 统一使用 `AlibabaPuHuiTi` 字体，确保清晰度与风格统一。
  - **同步精度**: 字幕显示时间需比旁白开始时间提前 0.1s，结束时间滞后 0.1s，避免“字到人未到”或“人走字未消”。

## Quality Checklist (必须可执行)
- [ ] **叙事连贯性**: 检查视频是否完整呈现“神秘→繁忙→震撼→温情”的情感曲线，无逻辑断层。
- [ ] **转场合理性**: 确认转场类型是否与场景情绪匹配（如：高潮前是否使用了 `zoom_in` 或硬切，而非柔和淡入）。
- [ ] **音画同步率**: 随机抽取 3 个旁白段落，检查文字出现时刻是否与画面关键动作（如车灯亮起、人群涌动）严格对齐（误差<0.2s）。
- [ ] **留白控制**: 确认旁白总时长占比在 20%-30% 之间，避免“念经式”配音，确保画面有呼吸空间。
- [ ] **技术规格**: 导出分辨率 1080p，时长控制在 60s±5s，无黑边、无音画不同步、无字幕遮挡关键主体。
- [ ] **素材多样性**: 确认素材包含至少 3 种不同视角（全景、中景、特写）和 2 种以上时间维度（日落、深夜）。

## Common Failure Patterns & Fixes
- **失败模式 1: 旁白时长严重不匹配**
  - **现象**: 系统提示“旁白明显短于分配时段”，导致画面静止等待配音，或配音语速过快。
  - **修复策略**: 触发“文案迭代机制”。若 `actual_narration_seconds` < `target_slot` * 0.6，自动扩充文案细节（增加形容词、副词）或拆分长句；若过长，则精简文案或缩短视频片段。
  - **本次案例**: 从 v1 (2.16s/5s) 迭代至 v2 (3.20s/5s)，虽仍有差距但已优化，最终通过留白策略接受 27% 覆盖率。

- **失败模式 2: 视觉节奏单一**
  - **现象**: 所有片段均为 5 秒，转场均为淡入淡出，导致视频像 PPT 演示。
  - **修复策略**: 强制引入“节奏变化器”。在 `plan_transition_timeline` 阶段，根据 `analyze_video` 的情绪标签，强制插入 20% 的 `zoom_in` 或 `hard_cut` 转场，并调整片段长度（3s-7s 随机分布）。

- **失败模式 3: 素材同质化**
  - **现象**: 搜索返回的素材多为同一角度的车流，缺乏叙事层次。
  - **修复策略**: 在 `search_bilibili_video` 阶段增加“负向关键词”或“视角限定词”（如：`-static`, `aerial view`, `subway interior`），并在 `rank_video_candidates` 中提高“视角多样性”的权重。

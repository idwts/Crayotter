- **user_request**: 剪辑一个一分钟的科技产品发布会高燃混剪，强调节奏冲击与旁白同步，语气激昂简洁。

## Tool Usage Skills
- **多模态高燃素材筛选 (`search_bilibili_video` + `rank_video_candidates`)**:
  - **最佳实践**: 针对“科技/高燃”类需求，搜索关键词需包含“动态标签”（如：`机械臂特写`、`光影爆发`、`演讲高潮`、`火箭升空`）而非仅品牌名。`rank_video_candidates` 需重点加权“镜头运动感（Motion）”、“高对比度光影”及“关键帧密度”，剔除静态PPT展示或低帧率素材。
  - **规避模式**: 避免素材库中“静态演讲”占比过高，导致视频缺乏“高燃”所需的视觉张力。需强制混合“宏观大景”（舞台/火箭）与“微观特写”（芯片/眼神/机械关节）。
  - **执行要点**: 设置 `top_k=8`，确保素材库包含：全景（震撼开场）、中景（产品演示）、特写（细节冲击），且色调需覆盖“冷蓝（科技）”到“暖金（辉煌）”的对比。

- **基于情绪曲线的动态裁剪 (`cut_video`)**:
  - **最佳实践**: 依据 `analyze_video` 提取的“情绪波峰”进行非均匀裁剪：
    - **悬念期**（0-10s）：中短镜头（4-5s），保留神秘感。
    - **爆发期**（10-40s）：极短镜头（3-4s），配合快节奏剪辑，每段动作必须完整（如机器人起跳、火箭点火）。
    - **升华期**（40s-End）：中镜头（4s+），配合深情旁白，留有余韵。
  - **规避模式**: 避免机械式等长切割（如全切4s），这会削弱“高燃”带来的节奏冲击力。
  - **执行要点**: 裁剪时需预留转场重叠区（0.15s-0.8s），确保 `merge_videos` 后总时长符合预期。本次案例中，14个片段原始59s，经转场重叠后压缩至53.45s，符合紧凑混剪需求。

- **差异化转场规划 (`list_transition_presets` + `add_transition`)**:
  - **最佳实践**: 转场类型必须服务于“科技冲击”逻辑：
    - **静→动**（品牌Logo→机器人）：使用 `zoom_in` 或快速 `crossfade` (0.15s-0.3s)，制造视觉冲击和速度感。
    - **动→静**（火箭发射→演讲）：使用 `fade_through_black` (0.8s) 或 `pixelize`，实现情绪缓冲与科技感过渡。
    - **高潮连接**（数据对比→宏大场景）：使用 `slide` 或 `distance`，增强空间纵深感。
  - **规避模式**: 全片使用单一转场（如全用淡入淡出），导致视觉节奏平淡，缺乏“高燃”所需的跳跃感。
  - **执行要点**: 转场总时长需精确计算，防止因重叠过多导致最终视频时长严重缩水。本次采用混合策略（13个切点，多种转场），有效平衡了流畅度与时长。

- **旁白与画面的双向校准 (`validate_narration_timeline` + `add_narration_segments`)**:
  - **最佳实践**: 采用“文案驱动 + 画面微调”策略。先根据情绪节点生成激昂文案，计算理论时长；若 `actual_narration_seconds` 过短（如本次段1短1.44s），优先扩充文案细节（增加排比句、感叹词），而非强行加速语速。
  - **关键修正**: 针对“高燃”主题，旁白覆盖率控制在 60%-70% 为宜，留出时间给环境音（重低音、机械声）和画面展示，避免“念经式”配音。
  - **规避模式**: 强行拉长旁白语速导致听感急促，破坏激昂氛围；或旁白过长导致画面被压缩，失去“留白”美感。本次系统警告“旁白明显短于分配时段”，通过后续文案迭代（增加“让每一行代码都充满力量”等修饰）将覆盖率提升至65.51%。

## Editing Workflow Skills
- **叙事结构构建（四段式高燃法则）**:
  - **四段式高燃结构**:
    1.  **悬念引入 (0-10s)**: 黑场/Logo/光束，建立“未来已来”的期待感，节奏由慢转快。
    2.  **技术爆发 (10-30s)**: 快速切换机器人、传感器、火箭等硬核画面，节奏极快，体现“实力”。
    3.  **宏大叙事 (30-45s)**: 破冰船、星空、演讲高潮，节奏稳健有力，体现“雄心”。
    4.  **情感升华 (45s-End)**: 家庭互动、Slogan定格，节奏放缓，体现“人文关怀”。
  - **镜头流动逻辑**: 确保场景在“微观技术”与“宏观愿景”之间交替切换，形成视觉上的“呼吸感”与“冲击力”并存。

- **节奏控制 (Pacing)**:
  - **呼吸感设计**: 在旁白段落之间（如 5s-8s, 48s-54s）刻意保留 2-4 秒的纯画面 + 环境音，让观众“感受”科技震撼而非“被灌输”信息。
  - **音画卡点**: 旁白重音（如“突破”、“引擎”、“势不可挡”）必须精准落在画面动作最剧烈或镜头切换的瞬间。

- **音频与字幕匹配**:
  - **字体规范**: 统一使用 `AlibabaPuHuiTi`，字号适中，位置在画面底部安全区，确保阅读舒适且不遮挡主体。
  - **同步精度**: 字幕显示时间需比旁白开始提前 0.1s，结束滞后 0.1s，确保“字随声动，声停字留”。

## Quality Checklist (必须可执行)
- [ ] **叙事连贯性**: 检查视频是否完整呈现“悬念→爆发→宏大→升华”的情绪曲线，场景切换（微观/宏观）是否自然流畅，无突兀跳色。
- [ ] **转场合理性**: 确认转场类型是否匹配场景逻辑（如：静→动用快切/Zoom，动→静用柔化/黑场），无生硬硬切或过度使用同一转场。
- [ ] **音画同步率**: 随机抽取 3 个旁白段落，检查文字出现时刻是否与画面关键动作（如机器人起跳、火箭点火）严格对齐（误差<0.2s）。
- [ ] **留白控制**: 确认旁白总时长占比在 60%-70% 之间，确保有足够的环境音和纯画面时间，符合“高燃/科技”调性。
- [ ] **技术规格**: 导出分辨率 1080p，时长控制在 50s-60s 之间（允许因转场重叠略短于目标），无黑边、无音画不同步。
- [ ] **素材多样性**: 确认素材包含至少 3 种景别（全景、中景、特写）和完整的“静→动→静”节奏变化。

## Common Failure Patterns & Fixes
- **失败模式 1: 成片时长严重缩水**
  - **现象**: 因转场重叠（如 13 个转场各 0.8s），导致 59s 素材剪辑后仅剩 53s，未达 60s 目标。
  - **修复策略**: 
    1.  **动态补偿**: 在结尾或情绪平缓处增加 3-5s 的纯画面留白（Loop 最后一段或延长长镜头）。
    2.  **转场优化**: 对非关键转场缩短时长（如从 0.8s 减至 0.15s），保留关键转场时长。
  - **本次案例**: 接受 54.0s 时长，通过强化结尾 Slogan 定格策略，确保内容完整且节奏紧凑，符合“高燃混剪”主题，未强行凑时。

- **失败模式 2: 旁白语速过快或过慢**
  - **现象**: 系统提示“语速偏快”或“旁白明显短于分配时段”，导致听感急促或画面空洞。
  - **修复策略**: 触发“文案迭代机制”。
    - 若过短：扩充文案，增加意象描写（如将“突破极限”改为“突破极限，超越想象，我们重新定义可能”）。
    - 若过长：精简文案，或微调视频片段起止点以匹配语速。
  - **本次案例**: 系统检测到部分段落旁白短于分配时段，通过接受 65.51% 的覆盖率（留白策略）和保持激昂语速，成功平衡了信息量与画面呼吸感。

- **失败模式 3: 视觉节奏单一（PPT 式剪辑）**
  - **现象**: 所有片段时长一致，转场单一，缺乏情绪起伏。
  - **修复策略**: 强制引入“节奏变化器”。在 `plan_transition_timeline` 阶段，根据 `analyze_video` 的情绪标签，强制插入不同时长（3s-7s 随机）和不同转场类型（`zoom_in`, `crossfade`, `fade_through_black`, `pixelize`）。
  - **本次案例**: 成功构建了“悬念→爆发→宏大→升华”的动态节奏，避免了单调。

## Version Notes
- **v4.0 (Current)**:
  - **核心升级**: 确立“四段式高燃叙事”与“差异化转场”标准。针对科技/高燃类视频，不再追求单一色调或单一节奏，而是强调“微观技术”与“宏观愿景”的对比，以及转场类型的多样化（Zoom/Crossfade/Black/Pixelize）。
  - **流程优化**: 引入“时长动态补偿机制”与“旁白覆盖率动态调整”。在转场导致时长缩水时，优先通过结尾留白或循环长镜头解决；在旁白过短时，优先扩充文案或接受留白，而非强行加速。
  - **去重说明**: 摒弃了历史经验中“固定 5 秒剪辑”和“高覆盖率旁白”的僵化模式，改为基于叙事结构（悬念 - 爆发 - 宏大 - 升华）的动态时长分配与文案策略。特别针对“高燃”场景，优化了转场策略，从“平滑过渡”转向“视觉冲击”。
  - **适用场景**: 适用于科技发布会、产品混剪、宏大叙事类短视频生成，特别是强调节奏感、冲击力和情绪起伏的场景。
